Configuring a WCAG-compliant color palette in Medhavy.md

Your Medhavy deployment reads its brand colors from Medhavy.md. If those hex values fail WCAG AAA contrast (7:1), every student and instructor using your platform sees text that is harder to read — and your institution carries the accessibility liability. This guide shows you how to use the Kitty tool to get compliant replacements through conversation, then commit them correctly to Medhavy.md.

Before you start — you need

Access to the Kitty palette tool (hosted at public/dev/kitty.html or the standalone URL your team uses).

Your current palette hex values — either from Medhavy.md directly, or by checking lib/theme.ts / app/globals.css if your deployment syncs from those files.

Write access to Medhavy.md in your repository. The changes you make here will propagate site-wide on next deploy.

Step 01 Understand what you are fixing and why

WCAG (Web Content Accessibility Guidelines) defines three contrast levels for text on background. The one that matters for an educational platform is AAA — a 7:1 ratio between foreground and background luminance.

Level Ratio Applies to Medhavy standard
AA 4.5:1 Normal text Minimum legal floor in most jurisdictions
AA Large 3:1 18px+ or 14px bold Not sufficient for body text
AAA 7:1 Normal text Required — Medhavy target
Why AAA for an educational platform?

Students read long-form textbook content, often on low-quality displays or under fluorescent lighting. Low contrast is not a minor inconvenience — it is a reading barrier that compounds over a 90-minute study session. Medhavy targets AAA because the platform's entire value proposition is access to knowledge.

The Medhavy default palette ships with eight colors (bb1–bb8). Several of those colors, used as text on the parchment background (bb8: #E8E0D0), fail AAA — and two fail AA entirely. The table below shows the default state before any upgrade.

Variable Hex Name Ratio on bb8 Status
bb1 #0D0D0D soot black 16.5:1 AAA ✓
bb2 #4A4A4A iron grey 5.9:1 Fails AAA
bb3 #8B0000 dried-ink red 5.2:1 Fails AAA
bb4 #8B7536 cold brass 4.5:1 Fails AAA
bb5 #2F2F2F charcoal 10.8:1 AAA ✓
bb6 #6B6B5E tarnished pewter ~3:1 Fails AA
bb7 #9C9680 aged ledger tan ~2:1 Fails AA
bb8 #E8E0D0 parchment — Background (fixed)

Step 02 Open Kitty and load your current palette

Kitty is a browser-based tool that combines a live contrast checker with a Claude-powered conversation. Open it at public/dev/kitty.html. When it loads, it will display the default Medhavy palette in the left panel, with auto-checked contrast ratios for all key pairs.

If your deployment uses a custom palette

If you have already modified the default colors, tell Kitty your current hex values in the chat. Say: "My palette is: bb1 #[hex], bb2 #[hex]…" Kitty will use those values for its analysis rather than the defaults.

The left panel auto-checks eleven key pairs — body text on parchment, accents on parchment, reversed text on dark backgrounds. Read through these before you start the conversation. The red FAIL badges are what you are fixing.

Step 03 Have the conversation with Kitty

Kitty will ask you about the brand before auditing the palette. Answer it. The context matters — a brand that needs to feel authoritative needs different fixes than one that needs to feel warm. Kitty's replacements are not mechanical darkening; they preserve the hue's emotional character while crossing the 7:1 threshold.

To get compliant replacement values, use this prompt:

You
My background is fixed at bb8 #E8E0D0. Suggest replacements for any colors that fail WCAG AAA against it. Stay close to the original hues — minimum necessary change only.
Kitty
Five colors need adjusting. bb2 from #4A4A4A to #434343 (7.1:1 — barely darker, same grey). bb3 from #8B0000 to #6B0000 (7.8:1 — deeper ink, still reads as the same red). bb4 from #8B7536 to #645420 (7.2:1 — loses a little warmth, still brass). bb6 to #4D4D43 and bb7 to #5E5A4A — both hit exactly 7.0:1. The muted feel is preserved; they are just usable as text now. bb1 and bb5 were already AAA, leave them.

If you want to verify a specific pair interactively, click two swatches in the left panel — the contrast checker will show you the ratio, a live text preview, and AA/AAA badges for both normal and large text sizes.

Do not accept replacements you have not reviewed visually

Contrast ratio is math. Brand feel is judgment. Look at Kitty's suggested hex values against the actual parchment background before committing. The left panel swatch grid lets you click any color to compare. If a replacement feels wrong, tell Kitty: "bb4 looks too olive — can you find something warmer that still hits 7:1?" It will iterate.

Step 04 Update the palette block in Medhavy.md

Once you have your compliant hex values, open Medhavy.md and find the palette block. It follows this structure:

## Color Palette

# Three files must stay in sync — edit all three together:
# lib/theme.ts · public/theme.json · app/globals.css

| Var | Hex     | Role                           | WCAG on bb8 |
|-----|---------|--------------------------------|-------------|
| bb1 | #0D0D0D | soot black — primary text      | 16.5:1      |
| bb2 | #4A4A4A | iron grey — primary accent     | 5.9:1       |
| bb3 | #8B0000 | dried-ink red — danger/emphasis| 5.2:1       |
| bb4 | #8B7536 | cold brass — highlight/callout | 4.5:1       |
| bb5 | #2F2F2F | charcoal — secondary accent    | 10.8:1      |
| bb6 | #6B6B5E | tarnished pewter — muted accent| ~3:1        |
| bb7 | #9C9680 | aged ledger tan — borders      | ~2:1        |
| bb8 | #E8E0D0 | parchment — page background    | —           |

Replace the failing hex values with your Kitty-verified replacements and update the ratio column. Using the default AAA upgrade, the block becomes:

| Var | Hex     | Role                           | WCAG on bb8 |
|-----|---------|--------------------------------|-------------|
| bb1 | #0D0D0D | soot black — primary text      | 16.5:1      |

| bb7 | #5E5A4A | aged ledger tan — borders      | 7.0:1        |

Highlighted rows show the changed values. bb1, bb5, and bb8 are unchanged — they were already compliant or are the fixed background.

The three-file rule

Medhavy.md is the source of truth, but three files must stay in sync: lib/theme.ts, public/theme.json, and app/globals.css. Update all three in the same commit. A palette split across versions will cause visual inconsistencies that are difficult to trace.

Step 05 Propagate changes to the three sync files

After updating Medhavy.md, apply the same hex values to the three files that drive actual rendering:

  1. Open lib/theme.ts. Update the hex values in the colors object. Update the comment from WCAG AA to WCAG AAA and add the ratio alongside each color.
  2. Open public/theme.json. Update the --bb-2 through --bb-7 values. Update the description field to reflect AAA compliance.
  3. Open app/globals.css. Update the CSS custom property values in the :root block.
  4. Commit all four files together: Medhavy.md, lib/theme.ts, public/theme.json, app/globals.css. A single atomic commit prevents the files from drifting out of sync on a partial push.
  5. Verify on deploy: open the site on a mobile device in direct sunlight if you can. AAA ratios should be readable. If anything looks off, return to Kitty with the specific pair that is failing.

Check How to verify the result

After deployment, verify compliance using any of the following:

  1. Return to Kitty with your updated hex values — the auto-checks panel should show all key pairs as AAA or AA, no FAIL badges.
  2. Run an automated audit using browser dev tools (Lighthouse > Accessibility) or a tool like axe DevTools. Every text element should report contrast passing.
  3. Check the ratio column in Medhavy.md manually. Every foreground color used for readable text should show 7.0:1 or higher against bb8.
What Kitty cannot check automatically

Kitty checks the palette colors against each other. It cannot check text rendered over a background image, text inside a gradient overlay, or text in components that apply opacity. Review those cases manually after the base palette update.
